Uploading Photos

PhotoHouse uploads photos directly to Amazon S3 using a resilient, browser-side queue. Files are split into chunks, progress is saved at every step, and uploads survive dropped connections, page refreshes, and closed tabs. You never lose work in progress.

How Uploading Works

When you add photos to the upload queue, PhotoHouse breaks each file into chunks and sends them in parallel directly to S3 — your photos never pass through the PhotoHouse server. This keeps uploads fast and avoids any server-side file size limits.

IndexedDB queue

The upload queue is stored in your browser's IndexedDB database — a persistent, local store that survives page closes and browser restarts. Every file in the queue has a record tracking its upload ID, which chunks have completed, and the current status (PENDING, UPLOADING, PAUSED, DONE, or FAILED).

Chunked multipart upload

Each file is divided into chunks (default 8 MB each). S3 receives the chunks and assembles them once all parts arrive. If a chunk fails, only that chunk is retried — not the whole file. This is especially valuable for large JPEGs and TIFFs over slow connections.

Auto-resume

When you return to the event page, PhotoHouse reads the queue from IndexedDB and resumes any interrupted uploads from exactly where they stopped — no re-uploading chunks that already succeeded.

No data loss

The queue is only cleared after a file reaches DONE status and its Photo record is confirmed in the database. A file is never removed from the queue mid-upload.

Selecting Photos to Upload

Open an event and click Upload Photos in the event header. The upload modal slides up from the bottom of the screen.

Drag and drop

Drag one or more image files directly onto the upload zone. The zone highlights when files are hovering over it. You can drag files from Finder (macOS), File Explorer (Windows), or any file manager on Linux.

Click to select

Click inside the upload zone or the Browse files button to open the system file picker. Hold ⌘ (Mac) or Ctrl (Windows) while clicking to select multiple files. To select an entire folder, right-click and choose Select all before dragging to the modal.

Selecting a group before uploading

If your event has photo groups, a Add to group dropdown appears in the upload modal. Select a group before clicking Upload and every file in the current batch is automatically assigned to that group when it finishes uploading. This saves you having to bulk-reassign photos after the fact.

💡
Upload in batches per group

Rather than uploading all ceremony and reception photos at once, upload the ceremony batch with "Ceremony" selected, then switch to the reception batch with "Reception" selected. Both batches can run in parallel.

Upload Progress

Once files are added to the queue, the upload modal switches to a progress view. Each file has its own row showing:

  • Filename — the original file name.
  • Progress bar — fills left-to-right as chunks upload. Colour changes from blue (uploading) to green (done) or red (failed).
  • Bytes transferred — e.g. 4.2 MB / 11.8 MB.
  • Status badge — UPLOADING, DONE, FAILED, or PAUSED.

Overall progress

At the top of the queue view, a summary bar shows how many files are done out of the total — e.g. Uploading… 14 / 38. This counter persists in the background tab so you can work elsewhere while uploads continue.

Retrying failed files

A file that fails after three retries shows a red Retry button. Click it to re-queue just that file without affecting the rest of the batch. Common causes of failure: file size exceeded your plan's storage limit, or the S3 presigned URL expired (happens if the tab was left open for more than 10 minutes before starting the upload).

If Your Internet Drops

🟢
PhotoHouse automatically pauses uploads when internet disconnects and resumes exactly where it left off when you're back online. You will never lose upload progress.

A network monitor watches for connectivity changes in real time. When a drop is detected, all in-flight chunk requests are paused gracefully. When the connection returns, a 2-second stabilisation delay runs before uploads resume — giving the connection time to settle before hammering S3 again.

You'll see an Offline banner appear at the top of the upload modal during the interruption, and a Resuming… message once the connection is restored. No files need to be re-selected.

Uploading from Mobile

The upload flow works on mobile browsers, though with some differences:

  • Tap to select — tapping the upload zone on iOS or Android opens the standard system picker. On iOS you can choose from your Camera Roll, iCloud Drive, or take a new photo directly. On Android you get Photos, Files, or Camera.
  • No drag-and-drop — dragging files is not supported on most mobile browsers. Use the tap-to-select flow instead.
  • Stay on the page — mobile browsers may suspend JavaScript when you switch to another app. If uploads pause unexpectedly, return to the PhotoHouse tab and they will resume within a few seconds.
ℹ️
Desktop is recommended for large batches

For events with hundreds of photos, uploading from a desktop browser on a wired or stable Wi-Fi connection is significantly faster and more reliable than mobile upload.

After Upload

Once a photo finishes uploading, PhotoHouse performs three background operations automatically — you don't need to trigger them manually.

Thumbnail generation

A compressed JPEG thumbnail (≈ 600 px wide) is generated using the Sharp image library and stored alongside the original in S3. The photo grid always loads thumbnails — not originals — keeping the grid fast even for 30 MB TIFF files. Originals are only fetched when a photo is opened in the lightbox.

EXIF data extraction

EXIF metadata is read from the uploaded file and stored in the database. The following fields are extracted when present:

  • Camera make and model (e.g. Canon EOS R5)
  • Focal length (e.g. 85 mm)
  • Aperture (e.g. f/1.8)
  • Shutter speed (e.g. 1/500 s)
  • ISO (e.g. 400)
  • Original capture date and time

This data appears in the lightbox info panel when you click the ⓘ button inside any photo.

Face detection (if enabled)

If Face Detection is enabled for the event and the platform allows new indexing jobs to start automatically, newly uploaded photos are queued for face analysis. Each detected face generates an embedding stored for future Find My Photos searches. See AI Face Search for the full details.
