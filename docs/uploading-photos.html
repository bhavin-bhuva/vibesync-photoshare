Uploading Photos

PhotoHouse uploads photos directly to Amazon S3 using multipart upload — meaning large files are sent in chunks and can resume after an interruption. The upload queue persists in your browser so you can close the tab and come back.

How to Upload

  1. Open an event and click "Upload Photos"

    The upload modal slides up from the bottom.

  2. Drag files or click to browse

    Select one or more JPEG, PNG, WebP, or TIFF files. You can select an entire folder.

  3. Optionally assign a group

    Pick a group from the dropdown before uploading to automatically file photos into that group.

  4. Click Upload

    Files are queued and uploaded in parallel. A progress bar shows bytes transferred per file.

Supported Formats

PhotoHouse accepts the following image formats:

  • JPEG / JPG — recommended for delivery (smallest file size)
  • PNG — for images with transparency
  • WebP — modern format, excellent compression
  • TIFF — archival quality; thumbnails are generated as JPEG
⚠️
RAW files not supported

CR2, NEF, ARW and other RAW formats are not accepted. Export from Lightroom or Capture One to JPEG or TIFF before uploading.

The Upload Queue

Every file you add enters a persistent browser-side queue backed by IndexedDB. This means:

  • You can close the upload modal and the queue keeps running in the background.
  • If you navigate away or the network drops, uploads resume automatically when you return to the page.
  • Each file shows its own progress bar and status: Pending → Uploading → Done.

Pausing and resuming

Use the pause button in the upload modal to temporarily halt the queue. Click resume to continue. Individual failed files can be retried with the retry button.

Thumbnails & EXIF

When a photo is saved, PhotoHouse automatically generates a compressed thumbnail (≈ 600 px wide) used for the photo grid. EXIF metadata — camera model, focal length, aperture, shutter speed, ISO, and capture date — is extracted and stored for display in the lightbox info panel.

Storage Limits

Your storage bar on the dashboard tracks total photo size across all events. When you approach your limit a warning banner appears. To see a breakdown, visit Dashboard → Storage in the admin panel.
