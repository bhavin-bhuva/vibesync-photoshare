# PhotoHouse Documentation

Standalone HTML/CSS/JS documentation site. No framework, no build step.
Open any `.html` file directly in a browser, or serve locally.

---

## Previewing the docs

```bash
# From the project root (inside Docker or locally if serve is installed)
npm run docs:serve
```

Opens at **http://localhost:4000**.

Or open any HTML file directly:
```
docs/getting-started.html
```

## Regenerating screenshots

Screenshots are captured by Puppeteer against a running local app.
Run this sequence every time the UI changes significantly.

### 1. Start the app

```bash
docker compose up -d
docker compose exec -T app npm run dev
```

Wait until you see `ready` in the dev server output.

### 2. Seed documentation test data

```bash
docker compose exec -T app npm run seed:docs
```

Creates:
- Photographer account: `docs@photohouse.test` / `DocsTest@123` (STUDIO plan)
- Admin account: `admin@photohouse.test` / `AdminDocs@123` (SUPER_ADMIN)
- 3 events with photos, groups, share links, and customer selections

Re-run safe — existing events are skipped if already present.

### 3. Run the screenshot suite

**Full suite (all sections):**
```bash
docker compose exec -T app npm run screenshots
```

**Individual sections:**
```bash
docker compose exec -T app npm run screenshots:auth
docker compose exec -T app npm run screenshots:dashboard
docker compose exec -T app npm run screenshots:events
docker compose exec -T app npm run screenshots:gallery
docker compose exec -T app npm run screenshots:settings
```

Screenshots are saved to `docs/screenshots/` and picked up by the HTML
pages automatically on the next page load.

### Screenshot naming convention

| Prefix | Page(s) |
|--------|---------|
| `auth-` | Login, register, pricing |
| `dashboard-` | Main dashboard, billing, profile |
| `event-` | Event detail: photos, people, selections, share, upload |
| `gallery-` | Public client gallery (share page) |
| `settings-` | Profile settings, admin panel pages |
| `admin-` | Admin panel pages |

Format: `{prefix}-{what}-{variant}.png`

Examples:
- `event-photos-tab-desktop.png` — event photos tab, 1440 px viewport
- `gallery-selection-mode.png` — gallery in photo selection mode
- `settings-billing-plan.png` — billing page showing current plan

## Adding a new documentation page

### 1. Create the HTML file

Copy an existing page (e.g. `uploading-photos.html`) and update:
- `<title>` tag
- Breadcrumb `<span class="breadcrumb-current">`
- `<h1 class="page-title">` and `<p class="page-intro">`
- Content inside `<div class="content-body">`
- Prev / Next navigation links at the bottom

### 2. Add it to the sidebar navigation

Add a `<li><a>` entry inside `<nav class="sidebar-nav">` in **every** HTML
file. The sidebar is repeated in each page (no server-side includes).

```html
<li><a href="your-page.html" class="nav-link">Your Page Title</a></li>
```

### 3. Add it to the search index in script.js

Append an entry to the `SEARCH_DATA` array:

```js
{ title: 'Your Page Title', url: 'your-page.html',
  keywords: 'relevant search terms here' },
```

### 4. Update prev/next links

Update the `page-nav` block on the preceding page (its `next` button)
and the following page (its `prev` button) to point to your new page.

### 5. Add screenshots

Create the capture function in `scripts/screenshots/capture-yourpage.ts`,
reference it from `capture-all.ts`, and add an npm script to `package.json`.

## Screenshot fallback

When a screenshot file doesn't exist, the `docs-screenshot` element shows:

> 📷 Screenshot pending — run `npm run screenshots` to generate

This is handled by the `onerror` event in `script.js` and the
`.screenshot-missing::before` CSS rule in `style.css`.
