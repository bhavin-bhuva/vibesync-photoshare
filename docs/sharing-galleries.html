Sharing Galleries

A shared gallery link gives clients private, branded access to your event photos. Each link has its own protection method, optional expiry date, and group visibility settings — so you can share exactly the right photos with the right people.

How Shared Links Work

When you create a share link, PhotoHouse generates a unique 16-character random URL slug — for example, /share/a3f8e2b91c047d6e. The URL is cryptographically random and not guessable by brute force, giving you a baseline of privacy even without a PIN or password.

Clients who visit the URL are authenticated by a short-lived, server-signed cookie. The cookie is scoped to the share link's slug, so accessing one gallery never grants access to another.

Creating a share link

  1. Open an event and click "Share Event"

    The Share modal opens, showing all existing links for this event.

  2. Choose a protection method

    Select No Protection, PIN, or Password from the toggle strip.

  3. Optionally set an expiry date

    Leave blank for a permanent link, or pick a calendar date.

  4. Click Create Link

    The new link appears in the list above with a one-click copy button.

  5. Copy and share

    Paste the URL into an email, WhatsApp, or SMS message to send to your client.

Customer experience

When a client opens your link they see a branded entry screen with your studio logo and name. After entering the PIN or password, they land on a gallery page styled with your brand colour and event cover photo in the hero header. All navigation (group filters, selection mode, download) is accessible from within the gallery — no login required.

Protection Options

No Protection

Anyone who has the URL can view the gallery immediately — no entry screen, no credential required. The URL itself is random and hard to guess, but there is no authentication layer.

Use this only for low-sensitivity content where you trust that the link won't be shared beyond the intended recipient.

PIN Protection Recommended

A 4-digit numeric PIN is the most client-friendly option. The entry screen shows four large input boxes; entering the PIN automatically submits — no button to tap.

You can share the PIN verbally at the end of a shoot: "Just tell your client: enter 4821." On mobile, the browser automatically opens the numeric keyboard when the PIN boxes are focused — no keyboard switching needed.

The plain-text PIN is stored in the database (alongside a bcrypt hash) so you can reveal it from the Share modal at any time if a client loses it.

💡
PIN protection is recommended for most clients

It's easy to share verbally at weddings and events, opens the numeric keyboard automatically on mobile, and clients can enter it without any typing confusion. Use a password only when you need a longer, more memorable credential — e.g. smithwedding2024 included in a welcome email.

Password Protection

A text password (minimum 4 characters, no maximum). Good when you want a meaningful phrase clients can remember without writing it down, or when you include the password directly in an email template.

Passwords are bcrypt-hashed before storage. The plain-text password is not stored, so if a client forgets it you'll need to revoke the link and create a new one.

Setting an Expiry Date

When creating a link, click the calendar icon next to Expires and pick a date. The link stops working at midnight on that date (UTC).

What happens when a link expires

Clients who visit an expired URL see a friendly "This gallery link has expired" screen with the event name. The entry screen does not show photos and cannot be accessed even with the correct PIN or password.

Expiry does not delete the event or its photos — only the share link becomes inactive. You can create a new link for the same event at any time.

ℹ️
No expiry by default

Leave the expiry field blank and the link remains active permanently — or until you revoke it manually. Most photographers set an expiry of 30–90 days to encourage clients to download promptly.

Controlling Downloads

Individual photo downloads

Clients can download individual photos using the download button in the photo lightbox. Each download request goes through a server endpoint that verifies the share cookie before streaming the file — direct S3 URLs are never exposed.

ZIP download

The Download All (ZIP) button lets clients download every visible photo in one archive. This feature requires Pro or Studio. On Free accounts, the button is shown but opens an upgrade prompt.

If a group filter is active, only the photos in the filtered group are included in the ZIP.

Watermarking on download

When watermarking is enabled in your Studio Profile, every downloaded photo — individual or ZIP — has your logo (or studio name) composited at the position and opacity you configured. The original S3 file is never modified; the watermark is applied server-side at download time. See Watermarking for setup details.

Watermarking applies on Pro and Studio only.

Group Visibility per Link

When you have multiple groups in an event, you can control which groups are visible on a per-link basis. This is set in the Share modal under Group visibility before creating the link.

Use case: staged gallery delivery

A common workflow for weddings:

  1. Upload all photos and organise into Ceremony, Portraits, and Reception groups.
  2. Create a first share link with only Ceremony and Portraits visible. Send this to the couple one week after the wedding for an early preview.
  3. When reception editing is complete, create a second link (or update visibility on the first) to reveal the Reception group.

This way you can release sections as they're ready without creating multiple events or waiting until everything is done.

Sharing with Your Client

Copying the link

After creating a link, click the Copy button next to the URL in the Share modal. The full URL is copied to your clipboard, ready to paste into any messaging app or email.

Via WhatsApp or SMS

Paste the URL directly into a WhatsApp message or SMS. Most smartphones render the URL as a tappable link. If you used PIN protection, include the PIN in the same message — for example:

Hi Sarah! Your wedding gallery is ready 🎉
Link: https://yourapp.com/share/a3f8e2b91c047d6e
PIN: 4821

What the client sees

Once through the PIN or password screen, clients see a full-screen gallery with:

  • Your studio logo and studio name in the hero header
  • The event cover photo (or your brand colour if no cover is set) as the hero background
  • The event name and date below the hero
  • Group filter pills if the event has multiple groups
  • Select Photos mode toggle for submitting selections
  • Download All button (Pro/Studio plans)
